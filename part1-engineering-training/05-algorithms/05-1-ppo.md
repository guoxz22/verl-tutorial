# 05-1 - PPO 训练

PPO（Proximal Policy Optimization）是 verl 支持的经典 RL 算法。

## 算法特点

- 使用 **Critic** 网络估计价值函数
- 使用 **GAE**（Generalized Advantage Estimation）计算 Advantage
- 支持 **KL 惩罚**（reward 或 loss 形式）
- 适合有 **Reward Model** 的场景

## 与 GRPO 的区别

| 特性 | PPO | GRPO |
|------|-----|------|
| Critic | 需要 | 不需要 |
| Advantage | GAE | Group 平均值 |
| 采样数 (n) | 通常 1 | 通常 > 1 |
| KL 惩罚 | Reward 中 | Loss 中 |
| 适用场景 | Reward Model | 可验证奖励 |

## 完整训练脚本

### PPO + Reward Model

```bash
#!/bin/bash
# run_ppo_rm.sh - PPO with Reward Model

set -x

# 数据路径
gsm8k_train_path=$HOME/data/gsm8k/train.parquet
gsm8k_test_path=$HOME/data/gsm8k/test.parquet
math_train_path=$HOME/data/math/train.parquet
math_test_path=$HOME/data/math/test.parquet

train_files="['$gsm8k_train_path', '$math_train_path']"
test_files="['$gsm8k_test_path', '$math_test_path']"

# 准备模型
hf download Qwen/Qwen2-7B-Instruct --local-dir $HOME/models/Qwen2-7B-Instruct &
hf download sfairXC/FsfairX-LLaMA3-RM-v0.1 --local-dir $HOME/models/FsfairX-LLaMA3-RM-v0.1 &
wait

python3 -m verl.trainer.main_ppo \
    algorithm.adv_estimator=gae \
    algorithm.gamma=0.99 \
    algorithm.lam=0.95 \
    algorithm.use_kl_in_reward=False \
    \
    data.train_files="$train_files" \
    data.val_files="$test_files" \
    data.train_batch_size=1024 \
    data.max_prompt_length=1024 \
    data.max_response_length=512 \
    data.filter_overlong_prompts=True \
    data.truncation=error \
    data.return_raw_chat=True \
    \
    actor_rollout_ref.model.path=$HOME/models/Qwen2-7B-Instruct \
    actor_rollout_ref.actor.optim.lr=1e-6 \
    actor_rollout_ref.actor.optim.lr_warmup_steps_ratio=0.1 \
    actor_rollout_ref.actor.ppo_mini_batch_size=256 \
    actor_rollout_ref.actor.ppo_micro_batch_size_per_gpu=16 \
    actor_rollout_ref.actor.use_kl_loss=False \
    actor_rollout_ref.model.enable_gradient_checkpointing=True \
    actor_rollout_ref.model.use_remove_padding=True \
    actor_rollout_ref.rollout.name=vllm \
    actor_rollout_ref.rollout.tensor_model_parallel_size=1 \
    actor_rollout_ref.rollout.gpu_memory_utilization=0.6 \
    actor_rollout_ref.rollout.n=1 \
    \
    critic.model.path=$HOME/models/Qwen2-7B-Instruct \
    critic.optim.lr=1e-5 \
    critic.optim.lr_warmup_steps_ratio=0.05 \
    critic.ppo_micro_batch_size_per_gpu=32 \
    critic.model.enable_gradient_checkpointing=True \
    critic.model.use_remove_padding=True \
    \
    reward.num_workers=8 \
    reward.reward_model.enable=True \
    reward.reward_model.model_path=$HOME/models/FsfairX-LLaMA3-RM-v0.1 \
    reward.reward_model.rollout.name=vllm \
    reward.reward_model.rollout.gpu_memory_utilization=0.8 \
    reward.reward_model.rollout.tensor_model_parallel_size=1 \
    \
    trainer.critic_warmup=0 \
    trainer.n_gpus_per_node=8 \
    trainer.nnodes=1 \
    trainer.total_epochs=15 \
    trainer.save_freq=20 \
    trainer.test_freq=5 \
    trainer.logger='["console","wandb"]' \
    trainer.project_name='verl_ppo_example' \
    trainer.experiment_name='qwen2_7b_rm'
```

### PPO + 函数奖励 + KL in Reward

```bash
#!/bin/bash
# run_ppo_function_rm.sh - PPO with Function Reward and KL penalty in reward

python -m verl.trainer.main_ppo \
    algorithm.adv_estimator=gae \
    algorithm.gamma=0.99 \
    algorithm.lam=0.95 \
    algorithm.use_kl_in_reward=True \
    algorithm.kl_penalty=kl \
    algorithm.kl_ctrl.type=adaptive \
    algorithm.kl_ctrl.kl_coef=0.02 \
    algorithm.kl_ctrl.target_kl=0.1 \
    \
    data.train_files=$HOME/data/gsm8k/train.parquet \
    data.val_files=$HOME/data/gsm8k/test.parquet \
    data.train_batch_size=512 \
    data.max_prompt_length=512 \
    data.max_response_length=1024 \
    \
    actor_rollout_ref.model.path=Qwen/Qwen2-7B-Instruct \
    actor_rollout_ref.actor.optim.lr=1e-6 \
    actor_rollout_ref.actor.ppo_mini_batch_size=128 \
    actor_rollout_ref.actor.ppo_micro_batch_size_per_gpu=32 \
    actor_rollout_ref.actor.use_kl_loss=False \
    actor_rollout_ref.model.enable_gradient_checkpointing=True \
    actor_rollout_ref.rollout.name=vllm \
    actor_rollout_ref.rollout.tensor_model_parallel_size=2 \
    actor_rollout_ref.rollout.n=1 \
    actor_rollout_ref.ref.fsdp_config.param_offload=True \
    \
    critic.model.path=Qwen/Qwen2-7B-Instruct \
    critic.optim.lr=1e-5 \
    critic.ppo_micro_batch_size_per_gpu=32 \
    \
    trainer.critic_warmup=10 \
    trainer.n_gpus_per_node=8 \
    trainer.total_epochs=30 \
    trainer.logger='["console","wandb"]' \
    trainer.project_name='verl_ppo_example' \
    trainer.experiment_name='qwen2_7b_function_rm_kl'
```

## 关键配置说明

### Algorithm 配置

```bash
# GAE 参数
algorithm.adv_estimator=gae      # 使用 GAE
algorithm.gamma=0.99             # 折扣因子
algorithm.lam=0.95               # GAE λ

# KL 惩罚（在 Reward 中）
algorithm.use_kl_in_reward=True
algorithm.kl_penalty=kl          # kl | abs | mse | low_var_kl
algorithm.kl_ctrl.type=adaptive  # adaptive | fixed
algorithm.kl_ctrl.target_kl=0.1  # 目标 KL（adaptive 模式）
```

### Critic 配置

PPO 需要 Critic 网络：

```bash
critic.model.path=Qwen/Qwen2-7B-Instruct
critic.optim.lr=1e-5
critic.ppo_micro_batch_size_per_gpu=32
critic.model.enable_gradient_checkpointing=True
```

### Trainer 配置

```bash
# Critic 预热：先训练 Critic 几步再训练 Actor
trainer.critic_warmup=10

# 采样配置
actor_rollout_ref.rollout.n=1    # PPO 通常 n=1
```

## 内存优化

### 启用 Offload

```bash
# Actor 参数 Offload（不推荐，影响性能）
actor_rollout_ref.actor.fsdp_config.param_offload=False

# Reference 模型 Offload（推荐）
actor_rollout_ref.ref.fsdp_config.param_offload=True

# Critic 参数 Offload
critic.model.fsdp_config.param_offload=True
```

### Gradient Checkpointing

```bash
actor_rollout_ref.model.enable_gradient_checkpointing=True
critic.model.enable_gradient_checkpointing=True
```

### 减小 Batch Size

```bash
data.train_batch_size=512
actor_rollout_ref.actor.ppo_mini_batch_size=128
actor_rollout_ref.actor.ppo_micro_batch_size_per_gpu=16
critic.ppo_micro_batch_size_per_gpu=16
```

## 常见问题

### 问题 1：Critic 不收敛

**症状**：`train/critic_loss` 不下降，`train/value_error` 很大

**解决**：
- 增加 Critic 预热步数：`trainer.critic_warmup=20`
- 降低 Critic 学习率：`critic.optim.lr=5e-6`
- 检查 Reward 是否合理

### 问题 2：KL 散度过大

**症状**：`train/kl` 持续增大，策略崩溃

**解决**：
- 增大 KL 惩罚系数：`algorithm.kl_ctrl.kl_coef=0.05`
- 使用 adaptive KL 控制
- 降低学习率：`actor_rollout_ref.actor.optim.lr=5e-7`

### 问题 3：OOM

**解决**：
- 启用 gradient checkpointing
- 减小 micro batch size
- 启用 Reference offload
- 使用 FSDP2 + CPU offload
